# yt-transcript

YouTube transcript/subtitle extractor for Node.js/Bun. Zero dependencies.

Uses the same approach as [yt-dlp](https://github.com/yt-dlp/yt-dlp): InnerTube `/player` API → `captionTracks` → `/api/timedtext` (json3 format).

## Why?

[youtubei.js](https://github.com/LuanRT/YouTube.js) cannot fetch transcripts because YouTube blocks the `/get_transcript` endpoint (HTTP 400, [issue #1102](https://github.com/LuanRT/YouTube.js/issues/1102)). This module bypasses that by using the caption tracks from the player response directly.

## Install

```bash
bun install
```

## CLI

```bash
# List available languages
bun run yt-transcript.ts dQw4w9WgXcQ --list

# Get transcript as JSON (default)
bun run yt-transcript.ts dQw4w9WgXcQ

# Get as SRT
bun run yt-transcript.ts dQw4w9WgXcQ --format srt

# Get as WebVTT
bun run yt-transcript.ts dQw4w9WgXcQ --format vtt

# Specific language
bun run yt-transcript.ts dQw4w9WgXcQ --lang en

# Full YouTube URLs work too
bun run yt-transcript.ts "https://www.youtube.com/watch?v=dQw4w9WgXcQ" --format srt
```

## Library API

```typescript
import { getTranscript, listTranscriptLanguages } from "./yt-transcript.ts";

// List available languages
const languages = await listTranscriptLanguages("dQw4w9WgXcQ");
// [{ code: "en", name: "English", isAutoGenerated: true }]

// Get transcript as structured JSON
const segments = await getTranscript("dQw4w9WgXcQ");
// [{ startMs: 840, endMs: 8599, text: "..." }, ...]

// Get as SRT string
const srt = await getTranscript("dQw4w9WgXcQ", { format: "srt" });

// Get specific language as VTT
const vtt = await getTranscript("dQw4w9WgXcQ", { lang: "en", format: "vtt" });
```

## Types

```typescript
interface TranscriptSegment {
  startMs: number;
  endMs: number;
  text: string;
}

interface Language {
  code: string;
  name: string;
  isAutoGenerated: boolean;
}
```

## How it works

1. POST to InnerTube `/youtubei/v1/player` with iOS client context (no auth/PO token required)
2. Extract `captionTracks` from `captions.playerCaptionsTracklistRenderer`
3. Fetch timedtext from track's `baseUrl` with `fmt=json3`
4. Parse `events[].segs[].utf8` with timing from `tStartMs`/`dDurationMs`
